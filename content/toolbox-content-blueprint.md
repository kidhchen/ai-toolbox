# AI制作工具箱内容蓝图

## 已确认方向

- 第一版为中文站。
- 安装包、HTML工具、Skill包、视频和图片后续统一上传到网站存储，不长期依赖外部文档附件。
- 评论区允许匿名昵称评论，用户可自己填写昵称；评论不做评分，改为文末像素风「好用」点赞。
- 网站风格走像素风、游戏化、极客工具站路线。
- 内容需要方便迭代，后续新增工具、替换安装包、更新视频不应依赖改代码。

## 网站定位

建议把网站包装成「AI制作工具箱」或「AI制作工坊」。它不是普通资料页，而是一个面向 AI 内容制作团队的工具补给站：

- 找工具：按制作任务快速找到视频、音频、素材、分镜、录屏和自动化方法。
- 学方法：每个工具都有说明、步骤、演示视频和适用场景。
- 用工具：提供在线入口、安装包下载或 Skill 使用方式。
- 给反馈：每个工具下方可匿名评论，并可点击「好用」点赞。
- 提需求：通过「许愿箱」收集制作痛点和新工具需求。

## 首页结构

首页建议采用游戏化工具大厅结构：

- 顶部区域：AI制作工具箱、搜索框、分类筛选、许愿箱入口。
- 工具分类：
  - 剧本分镜与提示词
  - 视频与音频制作
  - 图片与视觉素材
  - 自动化与工作流
  - 工具入口与管理
  - 教程与积木排版
- 工具卡片信息：
  - 工具名称
  - 一句话介绍
  - 工具类型
  - 开发者
  - 状态：已上线 / 开发中 / 持续迭代
  - 难度：入门 / 进阶 / 工作流
  - 最近更新时间
  - 进入按钮

## 工具详情页统一结构

每个工具详情页建议统一为以下模块：

1. 工具头部
   - 工具名称
   - 一句话介绍
   - 类型标签
   - 开发者
   - 当前版本
   - 状态

2. 适用场景
   - 适合谁用
   - 解决什么痛点
   - 推荐在什么环节使用

3. 核心功能
   - 3 到 6 个功能点

4. 使用步骤
   - 用清晰步骤说明如何安装、打开或运行

5. 资源入口
   - 在线预览
   - 安装包下载
   - 效果视频
   - 图文教程
   - Skill安装说明

6. 更新记录
   - 版本号
   - 更新时间
   - 更新内容

7. 评论区
   - 昵称
   - 评论内容
   - 是否遇到问题
   - 好用点赞

## 许愿箱

建议命名为「许愿箱」或「工具许愿池」，避免听起来像冷冰冰的反馈表。

页面文案：

> 说出你在 AI 制作里最想省掉的重复操作。我们会把高频痛点整理进工具箱，优先孵化成插件、HTML工具、Skill或自动化流程。

字段建议：

- 昵称
- 你遇到的制作痛点
- 希望它被做成什么形式：网页工具 / Chrome插件 / Codex Skill / 自动化脚本 / 不确定
- 当前你是怎么解决的
- 重要程度：一般 / 很想要 / 每天都卡
- 联系方式，可选

## 内容更新流程

建议后台支持以下动作：

1. 新增工具
2. 编辑工具说明
3. 上传安装包
4. 上传效果视频
5. 上传教程图片
6. 设置工具状态
7. 设置是否公开
8. 管理评论
9. 查看许愿箱提交

这样后续内容更新只需要进后台，不需要频繁改网站代码。
